Outil d’aide à la simulation visuelle de fonctionnement d’opérations de manipulation des structures de données statiques et dynamiques.

Les arbres binaires :

Un arbre binaire est une structure de données dynamique qui permet de stocker un nombre variable d’éléments de même type dans des nœuds, les éléments y sont représentés sous une hiérarchie ressemblant à celle d’un arbre où le nœud initial est appelé racine de l’arbre et où chaque élément possède au plus deux éléments fils au niveau inférieur. Parmi les opérations possibles sur un arbre binaire on peut trouver les suivantes : recherche, insertion, suppression, différents parcours (préordre, inordre, postordre)…

Arbres binaires

Les files :

Une file d’attente est une structure de données dynamique basée sur le principe dit FIFO (First In First Out) autrement dit : les premiers éléments ajoutés à la file seront les premiers à être récupérés illustrant le principe d’une file d’attente où les premières personnes arrivées dans la file sont les premières personnes à en sortir. Comme opérations de base sur une file, on trouve les opérations d’enfilement et de défilement.

File

Les piles :

Une pile est une structure de données dynamique basée sur le principe dit LIFO (Last In First Out) autrement dit : les derniers éléments ajoutés à la pile seront les premiers à être récupérés illustrant le principe d’une pile d’éléments quel qu’en que où les derniers à être ajoutés sur la pile sont les premiers à être récupérés. Comme opérations de base sur une pile, on trouve les opérations d’empilement et de dépilement

Pile

Les listes linéaires chainées :

Une LLC est une structure de données dynamique qui permet de stocker un nombre variable d’éléments de même type et ou chaque élément est repéré par un indice et pointe sur son suivant dans la liste. La liste linéaire chainée est dite triée si les éléments qu’elle contient sont ordonnés, non triée sinon. Selon qu’elle soit triée ou non, on distingue différentes opérations possibles sur une liste linéaire chainée : - Liste triée : chargement initial, recherche dichotomique, insertion (par valeur/par position), suppression (par valeur/par position)… - Liste non triée : chargement initial, recherche séquentielle (par valeur/par position), insertion (par valeur/par position), suppression (par valeur/par position), différents tris…

Liste linéaire chainée

Les matrices (tableaux bidimensionnels) :

Une matrice est un tableau à deux dimensions. Parmi les opérations possibles sur les éléments d’une matrice on peut trouver les suivantes : chargement initial, recherche d’un élément, produit matriciel, transposition et calcul du déterminant.

Matrice

Les tableaux (unidimensionnels) :

Un tableau est une structure de données statique qui permet de stocker un nombre fixé d’éléments de même type et où chaque élément est repéré par un indice. Le tableau est dit trié si les éléments qu’il contient sont ordonnés, non trié sinon. Selon qu’il soit trié ou non, on distingue différentes opérations possibles sur un tableau : - Tableau trié : chargement initial, recherche dichotomique, insertion, suppression. - Tableau non trié : chargement initial, recherche séquentielle, différents tris (tri par transposition, tri par insertion, tri bulles...)
